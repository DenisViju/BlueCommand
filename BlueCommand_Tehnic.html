Clean Architecture pe backend

Backend-ul urmează principiul Clean Architecture, împărțit în 4 proiecte cu dependențe unidirectionale.

ProiectRolConține
BlueCommand.Domain Nucleul aplicației — zero dependențe externe Entități (Dosar, Agent, Sectie…), Enum-uri (DosarStatus), Constante (RoluriDenumiri)
BlueCommand.Application Interfețe pentru servicii IAuditService, IJwtTokenService, IPasswordHasher, ICurrentUserService
BlueCommand.Infrastructure Implementări concrete (DB, servicii) BlueCommandDbContext, AuditService, BcryptPasswordHasher, Migrations, DatabaseInitializer
BlueCommand.API Stratul HTTP — entry point al aplicației Controllers, DTOs, Validators, Middleware, JwtTokenService, Program.cs
💡
De ce contează? Domain nu știe nimic despre baza de date sau HTTP. Infrastructure implementează interfețele din Application. API-ul depinde de Application (interfețe), nu de Infrastructure (implementări). Dependențele se injectează prin DI în Program.cs.

Conexiunea la PostgreSQL

EF Core se conectează la PostgreSQL prin Npgsql — provider-ul oficial. Connection string-ul este citit din fișierul .env sau appsettings.json.

C# · Program.cs înregistrarea DbContext în DI
// Preia connection string din .env sau appsettings.json var connectionString = builder.Configuration["DB_CONNECTION_STRING"] ?? builder.Configuration.GetConnectionString("DefaultConnection") ?? "Host=localhost;Port=5432;Database=bluecommand;..."; builder.Services.AddDbContext<BlueCommandDbContext>(options => { options.UseNpgsql(connectionString, o => o.EnableRetryOnFailure()); options.UseSnakeCaseNamingConvention(); // ex: NumarDosar → numar_dosar });
INI · appsettings.json connection string implicit
{ "ConnectionStrings": { "DefaultConnection": "Host=localhost;Port=5432;Database=bluecommand;Username=postgres;Password=postgres" } }

UseSnakeCaseNamingConvention

Proprietatea C# NumarDosar devine automat coloana numar_dosar în PostgreSQL. Fără mapări manuale de nume.

EnableRetryOnFailure

EF Core reface automat conexiunea dacă PostgreSQL nu răspunde temporar — rezistență la căderi tranzitorii de rețea sau restart server.

Autentificare JWT

Sistemul folosește JSON Web Tokens cu semnătură HMAC-SHA256. Token-ul este emis la login și trimis cu fiecare cerere HTTP în header-ul Authorization.

1. Login
POST /api/auth/login
→
2. Verificare
BCrypt.Verify()
→
3. Emitere
JwtTokenService
→
4. Stocare
localStorage (FE)
→
5. Cereri
Bearer {token}
C# · JwtTokenService.cs generarea token-ului
public string CreateTokenForUser(int userId, string username, string rolDenumire, int? sectieId, DateTime expiresUtc) { var claims = new List<Claim> { new("userId", userId.ToString()), new("username", username), new(ClaimTypes.Role, rolDenumire), // pentru [Authorize(Roles=...)] new("role", rolDenumire), // pentru frontend (jwt-decode) }; if (sectieId.HasValue) claims.Add(new Claim("sectieId", sectieId.Value.ToString())); // sectieId în token → filtrare automată dosare pe secție var key = new SymmetricSecurityKey(Encoding.UTF8.GetBytes(_secret)); var creds = new SigningCredentials(key, SecurityAlgorithms.HmacSha256); var token = new JwtSecurityToken( issuer: _issuer, // "BlueCommand" audience: _audience, // "BlueCommandClient" claims: claims, expires: expiresUtc, // 8 ore implicit signingCredentials: creds ); return new JwtSecurityTokenHandler().WriteToken(token); }
C# · Program.cs validarea token-urilor la fiecare cerere
builder.Services.AddAuthentication(JwtBearerDefaults.AuthenticationScheme) .AddJwtBearer(options => { options.TokenValidationParameters = new TokenValidationParameters { ValidateIssuer = true, // verifică "BlueCommand" ValidateAudience = true, // verifică "BlueCommandClient" ValidateLifetime = true, // verifică expirarea ValidateIssuerSigningKey = true, // verifică semnătura HMAC IssuerSigningKey = new SymmetricSecurityKey( Encoding.UTF8.GetBytes(jwtSecret)), ClockSkew = TimeSpan.FromMinutes(1) // toleranță 1 min }; }); // Politici de autorizare pe roluri builder.Services.AddAuthorization(options => { options.AddPolicy("AdministratorOnly", p => p.RequireRole("Administrator")); options.AddPolicy("SefSauAdmin", p => p.RequireRole("Administrator","SefInspectorat")); options.AddPolicy("TotiUtilizatorii", p => p.RequireAuthenticatedUser()); });
⚠️
Secretul JWT este citit din variabila de mediu JWT_SECRET (fișierul .env). În producție trebuie să fie un șir aleatoriu de minim 32 de caractere. Cel din appsettings.json este doar fallback pentru dezvoltare locală.
// 05 — backend · securitate

Hashing parole cu BCrypt

Parolele nu sunt stocate niciodată în clar. Se stochează exclusiv hash-ul BCrypt, care include salt aleatoriu generat automat.

C# · BcryptPasswordHasher.cs BlueCommand.Application/Security/
public class BcryptPasswordHasher : IPasswordHasher { // La înregistrare / setare parolă public string Hash(string password) => BCrypt.Net.BCrypt.HashPassword(password); // Produce ceva de genul: $2a$11$K5kV2...NXYabc... (60 caractere) // La login — compară parola introdusă cu hash-ul din BD public bool Verify(string password, string passwordHash) => BCrypt.Net.BCrypt.Verify(password, passwordHash); }

Ce este BCrypt?

Un algoritm de hashing adaptat pentru parole. Include un salt aleatoriu (prevenind rainbow table attacks) și un work factor (cost=11 implicit) care face brute-force extrem de lent.

Cum funcționează Verify()?

Extrage salt-ul din hash-ul stocat, aplică același algoritm pe parola introdusă și compară rezultatele în mod constant (fără timing attacks). Nu e nevoie să salvezi salt-ul separat.

Upload & servire fișiere

Documentele atașate dosarelor sunt salvate fizic pe server în wwwroot/uploads/{dosarId}/. Metadatele (nume, cale, mărime) sunt salvate în BD. Descărcarea este protejată prin middleware.

C# · DosareController.cs procesul de upload
// Extensii permise: .pdf .doc .docx .jpg .jpeg .png private static readonly HashSet<string> AllowedExtensions = new(...) { ".pdf", ".doc", ".docx", ".jpg", ".jpeg", ".png" }; // Validare mărime (max 10 MB) if (file.Length > 10 * 1024 * 1024) return BadRequest(new { error = "Fisierul depaseste 10 MB" }); // Construiește calea fizică var uploadsRoot = Path.Combine( Directory.GetCurrentDirectory(), "wwwroot", "uploads", id.ToString()); Directory.CreateDirectory(uploadsRoot); // creează folderul dacă nu există // Generează un nume unic (GUID) pentru a evita coliziunile var storedName = $"{Guid.NewGuid():N}{ext}"; // ex: a3f7...9b.pdf var storedPath = Path.Combine(uploadsRoot, storedName); // Salvează fizic fișierul await using (var stream = File.Create(storedPath)) await file.CopyToAsync(stream, cancellationToken); // Salvează metadatele în BD var doc = new DocumentDosar { DosarId = id, NumeFisier = file.FileName, // numele original CaleFisier = $"/uploads/{id}/{storedName}", // calea relativă MarimeBytes = file.Length, IncarcatDe = _currentUser.UserId ?? 0, DataIncarcare = DateTime.UtcNow };

Rapoartele pot fi exportate în două formate. QuestPDF generează PDF-uri, iar ClosedXML generează fișiere Excel (.xlsx). Ambele sunt returnate ca stream binar în răspunsul HTTP.

// QuestPDF — licență Community (gratuită) QuestPDF.Settings.License = LicenseType.Community; private static byte[] GenerateReportPdf(request, data) { var doc = Document.Create(container => { container.Page(page => { page.Margin(30); page.DefaultTextStyle(x => x.FontSize(12)); page.Header() .Text($"Raport: {request.Tip} ({request.DataStart})") .FontSize(16).SemiBold(); page.Content().Column(col => { col.Item().Text( JsonSerializer.Serialize(data)); }); }); }); return doc.GeneratePdf(); // returnează byte[] } // Răspuns HTTP return File(pdf, "application/pdf", $"raport_{request.Tip}_{DateTime.UtcNow:yyyyMMdd}.pdf");

// ClosedXML — wrapper fluent pentru OpenXML private static byte[] GenerateReportExcel(request, data) { using var workbook = new XLWorkbook(); var ws = workbook.Worksheets .Add(request.Tip); // nume foaie = tip raport ws.Cell(1, 1).Value = "Data"; ws.Cell(1, 2).Value = JsonSerializer.Serialize(data); ws.Columns().AdjustToContents(); using var ms = new MemoryStream(); workbook.SaveAs(ms); return ms.ToArray(); // returnează byte[] } // Răspuns HTTP cu MIME type corect return File(xlsx, "application/vnd.openxmlformats-...", $"raport_{request.Tip}.xlsx");

Axios cu interceptori

Toate cererile HTTP din frontend trec printr-o instanță Axios configurată cu 2 interceptori: unul care adaugă automat token-ul JWT la fiecare cerere, și unul care gestionează expirarea sesiunii.

TypeScript · src/api/axios.ts instanța Axios centrală
import axios from 'axios' const baseURL = (window as any).__API_URL__ ?? import.meta.env.VITE_API_URL ?? 'http://localhost:5001' export const api = axios.create({ baseURL }) // ① Interceptor REQUEST — adaugă JWT la fiecare cerere api.interceptors.request.use((config) => { const token = localStorage.getItem('bluecommand_token') if (token) config.headers.Authorization = `Bearer ${token}` return config }) // ② Interceptor RESPONSE — redirect la login dacă 401 api.interceptors.response.use( (res) => res, (err) => { if (err.response?.status === 401) { localStorage.removeItem('bluecommand_token') window.location.href = '/login' // redirect automat } return Promise.reject(err) }, )
💡
Avantaj: Nicio pagină nu trebuie să gestioneze manual token-ul sau erorile 401. Tot codul din src/api/ importă această instanță api și beneficiază automat de autentificare și logout automat la expirare.

'' setUser({ id: Number(decoded.userId), username: decoded.username, rol: mapRole(role), sectieId: decoded.sectieId ? Number(decoded.sectieId) : null, }) } catch { /* token invalid → logout */ } }, [token])

Formulare cu React Hook Form + Zod

Formularele sunt gestionate cu React Hook Form (performanță, fără re-render la fiecare tastă) și validate cu Zod (scheme TypeScript-first). Legătura dintre ele se face prin @hookform/resolvers.

TypeScript · DosarFormPage.tsx exemplu formular cu validare Zod
import { useForm } from 'react-hook-form' import { zodResolver } from '@hookform/resolvers/zod' import { z } from 'zod' // Schema Zod — tipuri + validare într-o singură definiție const schema = z.object({ numarDosar: z.string().min(1, 'Câmp obligatoriu').max(50), sectieId: z.number({ required_error: 'Alege o secție' }).positive(), dataIncident: z.string().optional(), tipIncident: z.string().optional(), }) type FormData = z.infer<typeof schema> // TypeScript type din schema const { register, handleSubmit, formState: { errors } } = useForm<FormData>({ resolver: zodResolver(schema) }) // Câmp cu eroare inline <input {...register('numarDosar')} /> {errors.numarDosar && <span>{errors.numarDosar.message}</span>}

React Hook Form

Nu stochează valorile în state React — operează direct cu DOM-ul prin ref-uri. Rezultat: zero re-render-uri la fiecare tastă, performanță maximă chiar și pentru formulare complexe.

Zod

Schema definește simultan tipul TypeScript și regulile de validare. Prin z.infer<typeof schema>, tipurile sunt deduse automat — fără duplicare de cod.
